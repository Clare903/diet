# 断食追踪

根据你上一餐吃的内容，智能计算断食各阶段的到达时间。

**在线使用：https://clare903.github.io/diet/**

手机打开后"添加到主屏幕"可当独立 App 使用，支持离线。

## 功能

- **饮食记录** — 两种模式：精确输入食物+克重（内置 40+ 常见食物数据库），或快速估算碳水和份量
- **智能阶段计算** — 根据碳水摄入量、进食量、断食经验，动态计算消化→血糖稳定→糖原消耗→燃脂→酮症→深度酮症的时间线
- **运动打卡** — 12 种运动类型，基于 MET 值和实际体重计算热量消耗，不同断食阶段运动效果不同
- **历史统计** — 记录每次断食的时长、达到的阶段、体重变化和感受

## 原理

断食进程主要受以下因素影响：

| 因素 | 影响 |
|------|------|
| 碳水摄入量 | 决定糖原储备量，碳水越多进入燃脂/酮症越慢 |
| 进食量 | 影响消化时间和糖原补充量 |
| 运动 | 加速糖原消耗，不同阶段效果不同（糖原消耗期效果最大） |
| 断食经验 | 经常断食的人身体更快切换到脂肪供能 |

运动热量消耗基于 [Compendium of Physical Activities](https://sites.google.com/site/compendiumofphysicalactivities/) 的 MET 标准值计算。

## 技术

纯前端，单个 HTML 文件，无需后端。数据存储在浏览器 localStorage 中。
